# Drawing Module - Usage Examples

## Visual Guide to Drawing Features

This document provides practical examples of how to use the drawing module effectively.

---

## Example 1: Highlighting a Problem Area

**Scenario**: You notice unusual activity in Track 3 between timestamps 1000-1500 and want to highlight it.

**Steps**:
1. Click "🖊️ Draw" to enter drawing mode
2. Click 🎨 and select **Red** (#ff0000)
3. Set brush size to **8** for visibility
4. Draw a **circle** around the problem area in Track 3
5. Change to **Yellow** (#ffff00)
6. Draw an **arrow** pointing to the circled area
7. Click "📥 Export" to save

**Result**: PNG image showing the chart with red circle and yellow arrow highlighting the issue.

**Use Case**: Bug reports, incident documentation, team meetings

---

## Example 2: Comparing Two Time Periods

**Scenario**: You want to compare activity patterns between two different time ranges.

**Steps**:
1. Enter drawing mode
2. Select **Blue** (#0000ff)
3. Set brush size to **5**
4. Draw a **vertical line** at the start of Period 1
5. Draw a **vertical line** at the end of Period 1
6. Change to **Green** (#00ff00)
7. Draw vertical lines for Period 2 boundaries
8. Add labels by drawing arrows or boxes
9. Export the annotated chart

**Result**: Clear visual comparison of two time periods.

**Use Case**: Performance analysis, trend comparison, before/after studies

## Example 3: Marking Milestones

**Scenario**: You need to mark important milestones or deadlines on the chart.

**Steps**:
1. Enter drawing mode
2. Select **Magenta** (#ff00ff)
3. Set brush size to **10** (bold)
4. Draw **vertical bars** at each milestone timestamp
5. Change to **Black** (#000000)
6. Set brush size to **3**
7. Add small annotations next to each bar
8. Export with clear milestone markers

**Result**: Chart with prominent milestone indicators.

**Use Case**: Project management, deadline tracking, sprint planning

## Drawing Techniques Reference

### Basic Shapes

**Circle/Oval**:
- Start at top
- Draw clockwise
- Connect back to start
- Use larger brush (8-12) for visibility

**Rectangle/Box**:
- Draw four sides
- Keep corners tight
- Use medium brush (5-7)

**Arrow**:
- Draw shaft (straight line)
- Add V-shaped head
- Use varied brush sizes

**Underline**:
- Draw straight horizontal line
- Use thin brush (2-4)
- Place directly under item

### Color Usage Patterns

**Status Indicators**:
- 🔴 Red: Problems, errors, critical
- 🟢 Green: Good, complete, approved
- 🟡 Yellow: Warning, pending, review needed
- 🔵 Blue: Information, notes, neutral

**Category Coding**:
- 🟠 Orange: Category A
- 🟣 Purple: Category B
- 🔵 Cyan: Category C
- 🟣 Magenta: Category D

**Emphasis Levels**:
- ⚫ Black: Primary annotation
- 🔵 Blue: Secondary detail
- White (on dark): Corrections

### Brush Size Guidelines

**Fine Detail (1-3px)**:
- Small annotations
- Precise markings
- Detailed notes
- Fine arrows

**Standard (4-7px)**:
- General annotations
- Normal circles
- Standard lines
- Readable marks

**Bold Emphasis (8-12px)**:
- Major highlights
- Important markers
- Attention grabbers
- Headers

**Very Bold (13-20px)**:
- Backgrounds
- Large shapes
- Cover-ups
- Major emphasis

## Best Practices Learned

### Do's ✅

1. **Plan Before Drawing**
   - Review chart first
   - Decide what to highlight
   - Choose color scheme

2. **Use Color Purposefully**
   - Consistent color meanings
   - Don't mix random colors
   - Limit to 3-4 colors per chart

3. **Keep It Simple**
   - Less is more
   - Focus on key points
   - Avoid cluttering

4. **Test Export Early**
   - Export test version first
   - Verify quality
   - Check readability

5. **Layer Information**
   - Start with major items
   - Add details progressively
   - Build complexity gradually

### Don'ts ❌

1. **Don't Over-Annotate**
   - Too many marks confuse
   - Keep space visible
   - Focus on essentials

2. **Don't Use Too Many Colors**
   - Stick to 3-4 colors
   - More colors = more confusion
   - Maintain consistency

3. **Don't Draw Too Fast**
   - Slow movements = smooth lines
   - Rushed drawings look messy
   - Take your time

4. **Don't Forget to Export**
   - Export before clearing
   - Save multiple versions
   - Keep originals

5. **Don't Block Important Data**
   - Draw around data, not over it
   - Keep labels visible
   - Maintain readability

## Troubleshooting Common Drawing Issues

### Issue: Lines are jagged
**Solution**: Draw more slowly, increase brush size slightly

### Issue: Can't draw where I want
**Solution**: Ensure you're in drawing mode, check cursor is crosshair

### Issue: Colors don't show well
**Solution**: Choose contrasting colors, avoid light colors on white background

### Issue: Export is too large/small
**Solution**: Export captures chart as-is, adjust chart zoom before exporting

### Issue: Mistakes in drawing
**Solution**: Use white color to "erase" over mistakes, or clear and redraw
